MediaKraken Multimedia Suite - How it works





How the MediaKraken server works.





How is metadata stored on server(s)

The metadata is stored into a 9.6 Postgresql database with json support. This allows new types of information to be stored with no db changes allowing faster implementation of newer features.


How it works:

  • The brains of the operation which all clients and slaves connect too.
  • This stores the central database which holds all the user and metadata.
  • Runs batch jobs to collect and process media and metadata.
  • Sends transcoding jobs to slave servers if CPU utilization suggests it.
  • Transcoding via FFMPEG
  • Web interface

Server Limits:

  • Total number of active and inactive users = 2,147,483,647
  • Active live TV channels = 5,000
  • Theoretical Limits
    • Movies, TV Shows, Songs, People, etc all use their own metadata table. Each table is limited to 32TB in size in Postgresql 9.5. This averages out to hundreds of millions of items each.
    • Records that point to the media file itself is limited to 32TB in size in Postgresl 9.5 as well. This should easily hold over 500 million items.
